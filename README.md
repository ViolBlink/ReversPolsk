# Калькулятор на основе обратной польской нотации

Консольный калькулятор на C++, который вычисляет математические выражения, переводя их в обратную польскую нотацию (ОПН) с помощью алгоритма сортировочной станции.

## Поддерживаемые операции

| Оператор | Действие |
|----------|----------|
| `+` | Сложение |
| `-` | Вычитание |
| `*` | Умножение |
| `/` | Деление |
| `^` | Возведение в степень (правоассоциативный) |
| `( )` | Скобки (вложенные поддерживаются) |
| `Sin(x)` | Синус (в радианах) |
| `Cos(x)` | Косинус (в радианах) |

Поддерживаются дробные числа (через `.`), унарный минус и переменные.

## Команды

Все выражения вводятся в виде команды с аргументом:

| Команда | Описание | Пример |
|---|---|---|
| `Calculate(expr)` | Вычислить выражение | `Calculate(2^10)` |
| `Show(expr)` | Показать ОПН-очередь | `Show(3+4*2)` |
| `Set(name, expr)` | Объявить переменную | `Set(x, 3+4)` |

## Сборка и запуск

### Требования
- CMake 3.15+
- Компилятор C++17 (GCC, Clang, MSVC)

### Сборка через CMake

```bash
cmake -S . -B build
cmake --build build
```

Бинарник будет помещён в папку `bin/`.

### Сборка в режиме Debug

```bash
cmake -S . -B build -DCMAKE_BUILD_TYPE=Debug
cmake --build build
```

### Запуск

```bash
./bin/main
```

## Алгоритм

1. `parce()` разбирает команду — определяет тип (`Calculate`, `Show`, `Set`) и извлекает аргументы.
2. Входная строка очищается от пробелов (`DeleteSpase`).
3. Унарный минус преобразуется в `0-` (`HandleNegative`), например `-3+1` → `0-3+1`.
4. `ToPolsk()` переводит выражение из инфиксной записи в ОПН по алгоритму сортировочной станции:
   - числа сразу попадают в выходную очередь;
   - функции (`Sin`, `Cos`) помещаются в стек операторов;
   - переменные подставляются по значению из словаря `Variables`;
   - операторы помещаются в стек с учётом приоритета и ассоциативности (`^` — правоассоциативный);
   - скобки обрабатываются рекурсивно.
5. `FromPolsk()` вычисляет ОПН-очередь: числа кладутся в стек, при встрече оператора/функции извлекается нужное количество операндов (из словаря `Arity`), результат кладётся обратно.
6. Ошибки (неизвестная переменная и др.) перехватываются в `main` через `try/catch`.
